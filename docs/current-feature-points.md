> 更新时间：2026-06-09

## 一、产品定位

论文助手是一个面向论文写作流程的 AI 工作台，采用“资料库 - 项目 - 阶段工作流”的结构。用户可以先在资料库沉淀长期材料，再进入具体论文项目，按材料理解、大纲确认、全文生成与修改的顺序完成论文草稿。

## 二、核心工作流

### 1. 资料库

- 支持手动添加文本资料。
- 支持上传 `txt`、`pdf`、`doc`、`docx` 类型资料。
- 支持资料标题、摘要、标签和类型管理。
- 支持风格标签提取，用于后续写作风格参考。
- 支持案例参考提取，用于后续写作时补充案例上下文。
- 支持将资料绑定到具体论文项目。

### 2. 项目管理

- 支持创建多个论文项目。
- 每个项目拥有独立标题、描述、阶段状态和上下文。
- 左侧栏展示项目入口，便于快速回到对应论文。
- 项目标题会同步到 Stage2、Stage3 和正文标题区域。
- 登录后项目数据会同步到 Supabase，未登录时保留本地 `localStorage` 兜底。

### 3. Stage1：材料理解

- 通过对话确认论文研究对象、写作边界、学段判断和核心论点。
- AI 在理解完成时输出结构化理解模型。
- 系统会从理解结果中提取或推断论文标题，并写入项目标题。
- 理解结果会保存进项目上下文，作为后续大纲和正文生成的核心背景。
- 已修正：理解完成时不再直接展示 JSON，而是展示可读的结构化结果。

### 4. Stage2：大纲生成与确认

- 基于 Stage1 的理解模型自动生成论文大纲。
- 大纲支持多级章节结构。
- 支持编辑章节标题。
- 支持新增子章节。
- 支持删除章节节点。
- 支持重新生成大纲。
- 支持确认大纲后进入全文生成阶段。
- 大纲数据支持同步到 Supabase 的 `outlines` 表。

### 5. Stage3：全文生成与修改

- 基于已确认大纲逐章生成完整正文。
- 正文区顶部显示论文总标题，和左侧项目标题保持同步。
- 支持重新生成全文。
- 支持选择章节后通过左侧对话提交修改意见。
- 支持 AI 改写、扩写、缩写、学术化等局部修改。
- 支持版本快照和恢复。
- 支持复制全文。
- 支持导出 Word `.docx`，可用 Word/WPS 打开。
- 新增“同步云端”按钮，用于手动将当前章节写入 Supabase `sections` 表并显示成功/失败结果。

## 三、AI 能力

- GPT/兼容 OpenAI API：用于材料理解、大纲生成、章节生成、摘要/引言/结语生成。
- 豆包/Volces Ark：用于文本改写、局部润色和框选修改。
- 支持流式输出。
- 支持上下文构建：项目理解、引用资料、已有章节、最近对话摘要等。
- 当前仍保留前端 AI 调用；正式上线前建议进一步将 AI key 完全下沉到后端 `/api/ai/stream`。

## 四、编辑器与导出

- 使用 `contenteditable` 实现正文直接编辑。
- 提供 Word/WPS 风格基础工具栏。
- 支持字体、字号、加粗、斜体、下划线、对齐等基础命令。
- 支持 Markdown 清洗，把 AI 输出中的 `###`、代码块、粗体符号等转成论文正文格式。
- 支持 `.docx` 导出，包含论文标题、章节标题和正文段落格式。

## 五、后端与数据同步

### 已完成

- Express 后端。
- Vercel API 入口。
- Supabase 服务端客户端。
- Supabase Auth 邮箱密码登录。
- Supabase PostgreSQL 表结构：
  - `user_profiles`
  - `projects`
  - `outlines`
  - `sections`
  - `versions`
  - `library_items`
  - `chat_messages`
  - `reference_selections`
- Supabase Storage bucket：`library-files`。
- 文件上传与解析入口。
- AI 服务端代理入口。
- 登录后远端同步，本地缓存兜底。

### 当前同步策略

- 未登录：全部数据保存在本地 `localStorage`。
- 已登录：启动时从 Supabase 拉取云端数据。
- 如果云端为空，会尝试将本地数据迁移到 Supabase。
- 保存时先写本地，再后台同步到 Supabase。
- 如果云端同步失败，不阻断前端使用。
