# 本周正式交付补齐清单

## 交付版定义

本周交付版不追求完整复刻 Word、Perplexity、SPSS 和问卷平台的全部能力，而是保证客户可以稳定完成一条真实论文工作流：

1. 登录并进入项目。
2. Stage1 识别题目、研究对象、学段和资料。
3. Stage2 生成并调整论文大纲。
4. Stage3 一键生成可编辑全文，失败可重试，过程可见。
5. 自动检索文献或后置引用增强，正文中有可解释引用，Word 导出保留脚注/参考文献。
6. 研究计算能沉淀资产：问卷、数据集、分析方案、图表、表格、论文表述。
7. 分析结果可一键写入论文对应章节。
8. 导出 Word 可打开、结构清楚、表图题和参考文献不乱。

## P0 必须补齐

### 1. 线上长任务稳定性

- 生成全文、文献准备、研究分析都不能表现为“点击后没反应”。
- 点击后必须立刻出现进度页或任务卡片。
- 失败时必须保留已有内容，并提供重新生成入口。
- 当前状态：已有 Stage3 进度页和研究分析进度，但仍主要依赖前端等待长请求。
- 建议补齐：增加统一任务状态记录，至少在本地/云端保存 `pending / running / failed / done`，支持刷新后恢复。

### 2. 云端数据闭环

- 项目、Stage1 聊天、Stage2 大纲、Stage3 正文、研究资产、研究包、上传文件、分析结果必须可恢复。
- 当前状态：Supabase + localStorage 混合，部分路径有本地兜底。
- 建议补齐：做一轮“换浏览器登录恢复”验收；缺表或缺同步字段马上补。

### 3. Word 导出验收

- 导出必须包含：标题、摘要、Abstract、章节编号、正文、脚注/参考文献、研究图表和表题。
- 当前状态：已有 docx 导出和多条 smoke 脚本，但需要用真实客户项目跑一遍。
- 建议补齐：固定一个客户演示项目，导出 docx 后人工打开 Word/WPS 检查。

### 4. 研究计算写入论文

- 上传数据后，用户看到数据集卡片。
- 确认分析方案后生成图表/表格/论文表述。
- AI 写入论文时，应自动选择研究方法、数据分析、结果讨论等章节。
- 当前状态：数据集卡片、方案确认、图表预览、写入论文链路已具备。
- 建议补齐：真实 Excel 样本跑 KANO/熵权、AHP、普通量化、定性编码各一次。

### 5. 登录和演示账号

- 正式站不能出现登录后无状态、循环跳转、点击无反应。
- 当前状态：有 `/login`、`/demo`、Supabase auth timeout 和本地兜底逻辑。
- 建议补齐：准备一个固定客户演示账号；登录后直接进入一个示例项目。

## P1 本周尽量补齐

### 1. 后置引用增强

- 先生成正文，再扫描可引用观点。
- 为每个观点匹配真实文献，插入脚注和参考文献。
- 不要把文献模块放成主流程负担。

### 2. 研究资产库体验

- 全局研究计算展示项目文件夹。
- 每个项目下展示多个问卷、多个数据集、多个分析结果、多个图表。
- 卡片显示状态：待收集、待分析、已分析、已写入论文。

### 3. 错误日志

- 记录接口、项目 ID、阶段、耗时、失败原因。
- 重点覆盖 `/api/ai/stream`、`/api/scholar/prepare`、`/api/research/analysis-plan`、`/api/research/analyze`。

### 4. 论文格式模板

- 默认使用中文论文格式：摘要、关键词、Abstract、Keywords、一级标题“一、二、三”、二级标题“（一）（二）”。
- 导出样式要和页面结构一致。

## P2 可以延期

- 真正 Word 级分页编辑引擎。
- 页底脚注挤占正文的精确排版。
- 图片/表格跨页复杂规则。
- 多人协作。
- 完整后台任务队列和管理后台。
- 复杂统计方法全覆盖，例如结构方程模型、PROCESS 全模型矩阵。

## 本周推荐执行顺序

1. 固定演示账号和演示项目。
2. 做一遍线上主链路冒烟：登录、Stage1、Stage2、Stage3、导出 Word。
3. 做研究计算真实样本冒烟：上传 Excel、确认方案、生成图表、写入论文、导出 Word。
4. 补长任务状态恢复和失败重试提示。
5. 补引用增强的后置入口。
6. 做最终 UI 文案清理，把“模块感”改成论文工作流语言。

## 当前验证证据

- `npx tsc --noEmit --pretty false` 通过。
- `npm run build` 通过。
- 当前风险不是编译失败，而是长任务稳定性、云端恢复、真实导出验收和结果质量。
