# 合同验收对齐矩阵

> 适用场景：已签合同后的交付自查、客户演示准备、尾款验收说明。  
> 原则：不写真实账号密码和密钥；不把增强功能伪装成合同必交功能；每一项都要能对应到可操作路径。

## 总体判断

当前系统已经具备 F1、F2、F3、F4、F5、F6、F7 的交付主链路。本文件用于交付前对照说明：哪些能力已经可演示，哪些属于增强边界或后续优化项。

1. **合同文字很具体，但产品采用了更自然的交互方式**  
   例如 F2 的 A/B 模式不需要额外按钮，可以解释为“新写作”和“基于原文修改”两条行为路径；F3 的自动调取目前用 `/` 主动调用风格档案来实现。

2. **F4 研究计算能力已补齐合同点名的核心统计入口**  
   目前已有量表/问卷、KANO、AHP、描述统计、Cronbach α、KMO、Bartlett、Pearson、T 检验、ANOVA、回归、中介、EFA、扎根/主题方向能力；分析结果可按论文结构拆成方法、图表、统计文字和讨论建议后插入正文。

3. **历史版本、导出、案例调用需要用产品说明限定边界**  
   系统能力比合同更强的地方也要说清楚，避免客户按“大型 SaaS / 多租户平台 / 自动检索一切资料”的标准验收。

## 演示建议

建议交付演示按这条线走：

1. 新建项目，Stage1 输入题目，展示材料理解和学段确认。
2. Stage2 生成大纲，确认论文结构。
3. Stage3 生成正文，演示选区 AI 改写。
4. 进入研究计算，展示系统读取题目、大纲、正文。
5. 选择“还没有数据”，生成问卷/量表。
6. 上传 Excel，生成分析方案，执行统计分析。
7. 一键把图、表、分析文字插入论文。
8. 导出 Word。
9. 展示历史版本和风格档案调用。
