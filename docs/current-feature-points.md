# 论文助手当前功能点

> 更新时间：2026-06-11  
> 当前版本：已接入登录、Supabase 云端存储、资料上传解析、三阶段论文工作流、TipTap 正文编辑器、Word 导出与 Vercel 线上部署。

## 一、产品定位

论文助手是一个面向论文写作流程的 AI 工作台，核心结构是“资料库 + 项目 + 三阶段工作流”。

它不是单轮聊天工具，而是让用户先沉淀资料，再为每篇论文创建项目，最后按“材料理解 - 大纲生成 - 全文撰写与修改”的顺序完成论文草稿。

## 二、当前推荐使用流程

1. 登录或注册账号。
2. 在资料库上传 PDF、Word、TXT，或手动存入文本资料。
3. 创建论文项目。
4. Stage1 上传或输入材料，让 AI 做材料理解。
5. 选择论文规格：本科、硕士或期刊。
6. Stage2 基于材料理解生成大纲，并可编辑、拖拽、增删章节。
7. 确认大纲后进入 Stage3。
8. 系统按大纲生成全文，用户在正文编辑器中继续修改。
9. 使用 AI 改写、缩短、扩写、学术化、脚注引用、版本历史、Word 导出等功能完成整理。

## 三、核心模块

### 1. 登录与云端同步

- 线上入口默认要求登录或注册。
- 使用 Supabase Auth 邮箱密码登录。
- 登录后项目、资料、章节、版本、引用选择等数据会写入 Supabase。
- 未登录或同步失败时，本地仍有 `localStorage` 兜底，避免前端直接阻断使用。
- 资料上传、正文解析和云端存储依赖登录状态。

### 2. 资料库

- 支持上传 `pdf`、`doc`、`docx`、`txt`。
- 上传后写入 Supabase Storage 与资料表。
- 后端解析文件正文，并保存 `text`、`summary`、结构提取、写法范式、观点线索、案例线索等信息。
- 支持手动添加文本资料。
- 支持资料标题、摘要、标签、类型管理。
- 支持资料绑定到具体论文项目。
- 资料可在 Stage1/Stage2/Stage3 通过引用面板进入 AI 上下文。
- “手动添加”独立按钮已弱化，用户主要通过写作框输入后存入库。

### 3. 项目管理

- 支持创建多个论文项目。
- 每个项目拥有独立标题、描述、当前阶段、绑定资料、项目上下文、章节和版本历史。
- 左侧栏展示项目入口，支持快速回到最近项目。
- 新对话会创建新的项目上下文，避免不同论文互相污染。
- 项目标题会同步到大纲、正文标题和导出文件。

## 四、三阶段工作流

### Stage1：材料理解

目标：先读懂用户提供的材料，而不是直接生成正文。

当前能力：

- 支持输入题目、材料、想法、摘要、提纲，或上传附件后理解。
- 上传文件会先上传解析，拿到正文、摘要和提取结果后，再进入材料理解 prompt。
- 正文摘录不再硬性截断为 7000 字，当前会把解析出的正文全文送入材料理解上下文。
- Prompt 已放宽：用户给什么就先读什么，不再强制追问研究对象、写作边界或学术边界。
- AI 输出材料理解结果，包括推荐题目、主题判断、可写方向、可展开论点、材料缺口/风险、建议写作难度。
- 不再输出“大纲生成建议”。
- 本科、硕士、期刊不再由 AI 自动猜测，理解完成后必须由用户在界面选择论文规格。
- 未选择论文规格前，不能进入大纲撰写。

Stage1 会写入项目上下文：

- `researchObject`
- `writingBoundary`
- `academicLevel`
- `rawSummary`
- `writingRequirements` 中的论文规格

### Stage2：大纲生成与确认

目标：把 Stage1 的材料理解转成论文结构母版。

当前能力：

- 基于 Stage1 理解模型生成大纲。
- 默认包含 `0 摘要`，用于后续生成中文摘要、关键词、英文 Abstract 和 Keywords。
- 大纲支持多级章节。
- 支持编辑章节标题。
- 支持新增章节和子章节。
- 支持删除章节。
- 支持拖拽移动章节，改变排序。
- 大纲变更会记录版本历史。
- 确认大纲后进入 Stage3。
- 进入 Stage3 时会有“大纲到正文”的动画过渡。

设计原则：

- 大纲是正文结构母版。
- 正文应跟随大纲变化。
- 顺序移动优先移动正文顺序，不重新生成。
- 新增章节只生成新增部分。
- 删除章节应先进入归档/确认逻辑，避免误删。
- 标题含义变化、合并/拆分章节等需要专门 AI 整理动作。

### Stage3：全文生成、编辑与修改

目标：按已确认大纲生成完整正文，并提供接近 Word 的编辑体验。

当前能力：

- 首次生成会先生成“全文写作计划”，再按一级章节逐章生成正文。
- 每章生成时带完整大纲、全文计划、项目理解和引用策略，降低章节断裂。
- 摘要章节会生成：
  - 中文摘要
  - 中文关键词
  - English Abstract
  - English Keywords
- 正文生成时按论文规格区分本科、硕士、期刊写法。
- 生成过程中显示进度条和章节进度。
- Stage3 若处于确认大纲后但正文尚未落地，会显示“正在准备正文”的 loading 状态，不再露出空文档裸状态。
- 支持重新生成全文。
- 支持按大纲新增章节补写新增部分。
- 支持选择章节后通过左侧对话提交修改意见。
- 支持版本快照和恢复。
- 支持复制全文。
- 支持同步云端。

## 五、正文编辑器

当前 Stage3 已从手写 `contentEditable` 迁移到 TipTap/ProseMirror 编辑器内核。

当前能力：

- Enter 换段。
- Backspace 合并段落。
- Ctrl+Z / Ctrl+Y 撤回与重做。
- 跨段选择、删除、复制、粘贴。
- 正文可随时编辑，不需要每次点击“手动编辑”。
- A4 宽度论文样式预览。
- 标题居中显示，不隐藏标题。
- 标题下方直接进入内容，减少不必要空隙。
- 编辑器 JSON 存入 `content_doc`，纯文本 `content` 继续保留用于兼容 AI、搜索和导出。

## 六、局部 AI 工具

正文中选中文本后，会出现基于 TipTap BubbleMenu 的浮动工具栏。

当前能力：

- AI 改写。
- 缩短。
- 扩写。
- 学术化。
- 添加脚注。
- 支持确认或取消 AI 修改。
- 工具栏位置针对长选区做过调整，避免因为工具栏太长而点不到确认按钮。

## 七、引用、脚注与参考文献

当前能力：

- 支持在 AI 生成和修改时引入资料库内容。
- 支持通过 `@` 引用资料维度。
- 生成正文时可按 `[1]`、`[2]` 类格式插入引用标记。
- 选区添加脚注后，正文选区绑定脚注标记。
- 脚注内容保存在章节 footnotes 中。
- 支持编辑脚注、删除脚注。
- 支持根据脚注生成参考文献区域。
- Word 导出会尽量保留标题、正文段落、脚注与参考文献。

## 八、版本历史

当前能力：

- AI 生成正文会记录版本快照。
- AI 修改正文会记录版本快照。
- 手动编辑正文会记录版本快照。
- 大纲变更会记录版本历史。
- 支持查看、预览和恢复旧版本。
- 删除章节、结构变化等高风险操作会尽量进入版本历史。

## 九、Word 导出

当前能力：

- 支持导出 `.docx`。
- 导出内容包括论文标题、摘要、章节标题、正文段落、脚注、参考文献。
- 导出优先读取编辑器 JSON；没有编辑器 JSON 时回退解析纯文本。
- 已修复动态导入导致的线上 Word 导出失败问题。

## 十、AI 模型使用

- GPT/兼容 OpenAI API：用于材料理解、大纲生成、全文计划、章节生成、摘要/引言/结语等。
- 豆包/Volces Ark：用于局部改写、缩短、扩写、学术化等文本处理。
- 支持流式输出。
- AI 上下文会组合项目理解、引用资料、当前章节、已有正文、最近对话和用户输入。

## 十一、后端与数据

当前后端能力：

- Express 后端。
- Vercel API 入口。
- Supabase Auth。
- Supabase PostgreSQL。
- Supabase Storage。
- 文件上传与解析。
- AI 服务端代理入口。
- RLS 策略已处理资料上传与项目数据写入。

主要数据表：

- `user_profiles`
- `projects`
- `outlines`
- `sections`
- `versions`
- `library_items`
- `chat_messages`
- `reference_selections`

重要字段：

- `sections.content`：兼容纯文本正文。
- `sections.content_doc`：TipTap/ProseMirror JSON 文档。
- `sections.footnotes`：脚注数据。

## 十三、当前待完善点

- Stage2 对缺少 Stage1 理解结果的项目，可以增加更明确的引导或 fallback。
- 合并/拆分大纲章节后，需要更完整的 AI 内容迁移流程。
- 删除章节的归档确认可以进一步产品化。
- AI 引用需要进一步提高来源匹配准确度和格式稳定性。
- Word 导出可继续细化学校格式、页眉页脚、目录、参考文献规范。
- 需要增加用户用量限制、AI 调用日志和更明确的隐私说明。
- 可继续拆分前端大 chunk，优化加载性能。
